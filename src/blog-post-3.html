Plutus Pioneer Week #1

Last Updated 05/02/2021

So if you are like me and have decided you wanted to learn Plutus, but were to late / not one of the 1500 people to be involved in this spring’s Plutus Pioneers Program(PPP) put on by IO-HK, then you are in luck, my blog posts will be out while / before you are working on the course. The PPP content is open and free on Github, with the rest of Cardano here. As someone who is new to programing there were quite a few things that went right over my head and I had to do lots of additional research to learn. I want to summarize it all below.


To begin I would watch Lecture #1 . After watching the lecture we can begin with the homework.


Homework #1 is focused on getting your system up and running and testing an example contract that is given to you. Plutus only runs in a Linux based environment which means that, if you are like me and have a windows based PC, you will need to download a virtual machine in order to work with the code.

Downloading the Virtual Machine

So, to download the virtual machine I highly recommend following this link to YouTube and follow along with the tutorial. There are a couple of specific catches that I want you to be aware of to ensure you do not run into any issues on further steps.

  • Set the Virtual storage space higher than 10 Gb. I failed to do this the first time and ran out of space, so I reccomend 100 Gb.
  • Enable Virtual machines in your BIOS: Press CTRL+SHIFT+ESC then Performance. There will be “Virtualization: ______” If its Enabled you’re good, if its Disabled you will need to restart your PC and update this in the BIOS. Look up a YT tutorial for your motherboard mfg. I have a gigabyte board and it was hidden deep in the settings and I would not have found it without help.

Setting up the VM

In the terminal we will first download Git by running the command sudo apt-get install git after this you will need to create a directory for Plutus by running the command mkdir Plutus At this point you are ready to clone the plutus repository into this directory. Run git clone https://github.com/input-output-hk/plutus.git


While windows uses ctrl + c/v/x , in linux you will use ctrl+shift + c/v/x and so on


To use nix (which is what we need to enter a nix-shell), we will download this software with a code found on the nix OS website curl -L https://nixos.org/nix/install | sh You’ll notice if you try to run this code right off the bat, you will get an error asking you to install curl with sudo apt install curl After this nix install you will need to cycle the power of your VM. To confirm that this worked, relaunch the VM -> terminal -> cd plutus -> nix this should show a long list of potential commands.

Setting up the IOHK Binary Caches

We will first set up a nix config file. In the terminal cd /etc then create a nix directory (note I needed to create this with administrative privileges so that’s why the sudo is needed) sudo mkdir nix Next we need to install the text editor vim so that we can add the code that is listed in the HW. sudo apt install vim After that has loaded, lets create a the config file by typing sudo vim nix.conf now in our file copy and paste the code from here. To save type :wq *enter*. Lets now get back over to the plutus directory cd ~ cd plutus


At this point we are ready to actually build the plutus core by running nix build -f default.nix plutus.haskell.packages.plutus-core (also found on the IOHK github) This will take some time, but after it is downloaded, run nix-shell then add a second terminal page (click the plus sign in the top left) and run nix-shell there as well.


Enter the plutus playground client cd plutus-playground-client/ then plutus-playground-server


Switch to the other terminal and start the playground client, but this time we will just start the client by npm run start After the local host has finished, you will need to scroll up to find the exact address. See image below:

Right click and open the link.


Ready to run an English Auction


Ok well, we are at least ready to now run the test code that is called English Auction. The original code will fail to compile, but the update in week 3 will succeed.
